# Contributing to πDMS

First of all — **thank you** for considering contributing to πDMS! 🙏

We’re building digital infrastructure that helps millions of people navigate bureaucracy. Every contribution — big or small — matters.

-----

## 🌟 Ways to Contribute

### 1. 🐛 Report Bugs

Found something that doesn’t work on [pidms.de/share.html](https://pidms.de/share.html)?

→ [Open a Bug Report](../../issues/new?template=bug_report.md)

Please include:

- What you expected to happen
- What actually happened
- Browser / device / OS
- Screenshots if possible

-----

### 2. 💡 Suggest Features

Have an idea that would make πDMS better?

→ [Open a Feature Request](../../issues/new?template=feature_request.md)

We especially welcome ideas around:

- Accessibility improvements
- New language support
- Authority / government integrations
- Mobile UX improvements

-----

### 3. 🌍 Translations

πDMS supports 8 languages — but we want more!

**Currently supported:**

- 🇩🇪 German (native)
- 🇬🇧 English
- 🇹🇷 Turkish
- 🇵🇱 Polish
- 🇷🇴 Romanian
- 🇦🇷 Spanish
- 🇷🇺 Russian

**We need help with:**

- 🇦🇪 Arabic
- 🇺🇦 Ukrainian
- 🇻🇳 Vietnamese
- 🇮🇹 Italian
- 🇫🇷 French
- Any other language spoken in Germany!

If you can help translate the AI prompt or UI strings, please open an issue or email kontakt@pidms.de

-----

### 4. 🎨 UI/UX Feedback

We’re building for **everyone** — including people who aren’t tech-savvy.

Your feedback on the current prototype is incredibly valuable:

- Is anything confusing?
- What could be clearer?
- How would your grandmother use this?

→ [Start a Discussion](../../discussions/new)

-----

### 5. 🔌 Technical Contributions (Flutter)

The native Flutter app is in planning phase. When development starts, we’ll need:

**Core areas:**

- Document scanning & OCR integration
- Local database (Isar) setup
- Supabase cloud sync
- Universal Bridge API client
- Push notifications
- Accessibility features

**Skills needed:**

- Flutter / Dart
- Riverpod state management
- REST API integration
- Local-first architecture

Want to be involved from day one? Email kontakt@pidms.de or open a Discussion.

-----

## 🤝 Code of Conduct

Be kind. Be constructive. We’re building something that helps real people.

- Respect all contributors regardless of background
- Constructive criticism is welcome — personal attacks are not
- Remember: our users include seniors, people with disabilities, and people who don’t speak German as their first language

-----

## ⭐ Recognition

All contributors will be:

- Listed in our CONTRIBUTORS file
- Mentioned in release notes
- Invited to early access when the native app launches

**Significant contributors** (especially early Flutter developers) may be eligible for:

- Founding team recognition
- Equity discussion for sustained contribution
